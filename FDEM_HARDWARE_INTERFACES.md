# FDEM 系统硬件接口说明

## 1. 文档目的

本文档说明 FDEM（频率域电磁法）系统中 PXI、功率放大器、发射线圈、接收线圈及监测回路之间的接口关系，并规定 `Signal in` 波形和操作安全要求。

本文档参考 `HARDWARE_INTERFACES.md` 中的 TEM 系统接口框架编写。FDEM 与 TEM 的核心区别是：

- TEM 的 `Signal in` 使用阶跃或矩形控制信号；
- FDEM 的 `Signal in` 必须使用严格零直流分量的正弦波；
- 每次 FDEM 发射包含 `n` 个频率为 `f` 的完整正弦周期；
- `Signal in` 当前配置为 `6 Vpk`（`12 Vpp`）；
- 功率放大器根据输入正弦波驱动发射线圈，输出对应频率的正弦波。

> **最高级别安全警告：FDEM 功率放大器的 IGBT 没有任何保护措施。`Signal in` 一旦出现直流信号或带直流偏置的正弦信号，IGBT 会立即烧坏。任何未经示波器验证的波形都不得接入功放。严禁将 TEM 使用的 `0 V -> 4 V` 阶跃波形输出到 FDEM 功放的 `Signal in`。**

> **参数限制：**当前资料没有给出功放型号、端子编号、允许频率范围、负载范围、增益、输入阻抗、最大直流偏置容限和输出额定值。`6 Vpk`（`12 Vpp`）是当前软件配置值，不等于已经确认的功放安全输入额定值。因此本文档确定接口用途和必要的安全约束，但不能代替功放、PXI 板卡及接线附件的硬件手册。

## 2. 系统组成

FDEM 系统由以下部分组成：

1. Mac 控制端：运行图形界面，通过 SSH 控制 PXI。
2. NI PXI 机箱：产生发射波形并同步采集监测信号。
3. PXIe-6361：通过模拟输出通道产生功放启动信号和 FDEM 正弦控制信号。
4. FDEM 功率放大器：放大 `Signal in` 的正弦信号并驱动发射线圈。
5. 发射线圈 Tx：产生频率为 `f` 的一次交变磁场。
6. 电流监测及衰减/隔离电路：测量发射回路电流。
7. 接收线圈 Rx：接收一次场耦合及目标二次场响应。
8. PXIe-4309：同步采集发射电流监测信号和接收线圈信号。

系统总体连接关系为：

```text
Mac GUI
   |
   | SSH / Ethernet
   v
PXI controller
   |
   +-- PXIe-6361 ao1 --------------------> Power amplifier Start
   |
   +-- PXIe-6361 ao0 -- zero-offset sine -> Power amplifier Signal in
   |                         n cycles, frequency f, amplitude 6 Vpk (12 Vpp)
   |
   +-- PXIe-6361 ai10 <------------------- Power amplifier Ready
   |
   +-- PXIe-4309 ai29 <------------------- Current-monitor voltage
   |
   +-- PXIe-4309 ai31 <------------------- Receiver signal

Power amplifier output --> Tx coil --> primary sinusoidal field
                                             |
                                             v
                                      underground target
                                             |
                                             v
                                      secondary field --> Rx coil
```

## 4. FDEM 功率放大器接口

### 4.1 `Start` 启动接口

连接关系为：

```text
PXIe-6361 ao1 signal/reference ----> power amplifier Start input/reference
```

`Start` 的结构、功能和用法与 TEM 功率放大器完全一致。它是低功率启动控制输入，不是 `Signal in`，也不是发射线圈功率输入。当前系统使用 `PXIe-6361/ao1` 输出启动波形，使功放开始充电或进入工作准备状态。

启动波形为 111 点有限输出波形：

- 采样率：`10,000 S/s`；
- 任务缓冲区约 `501.1 ms`（前置 `1 ms`、高电平 `500 ms`，加 1 个终止采样点）；
- 前 10 点：`0 V`，持续 `1 ms`；
- 中间 5000 点：`4 V`，持续 `500 ms`；
- 最后 1 点：`0 V`，明确结束高电平；
- 核心启动事件：`0 V -> 4 V` 上升沿；
- 空闲状态：`0 V`。

```text
ao1 voltage
4 V              +-------------------------------+
                 |
0 V -------------+                               +----
     0           1 ms                        501 ms
```

前置 `0 V` 用于建立确定的低电平，使功放能够识别清晰的上升沿。操作时先发送 `Start` 启动信号，观察功放状态并等待其完成充电或进入准备状态，然后才能通过 `Signal in` 发射 FDEM 正弦波。

启动波形末尾包含明确的 `4 V -> 0 V` 下降沿。由于 NI-DAQmx 任务结束后的物理端口状态仍取决于板卡和驱动，必须使用示波器确认任务结束后 `ao1` 确实保持 `0 V`，不得仅根据任务关闭推定端口状态。

> **接口区分：**`Start` 接口使用上述 `0 V -> 4 V` 保持 `500 ms` 再回到 `0 V` 的启动波形；FDEM 的无直流和无偏置要求只针对 `Signal in`。`ao1` 的启动波形不得误接到 `Signal in`。

### 4.2 `Signal in` 正弦波输入接口

#### 4.2.1 连接关系

```text
PXIe-6361 ao0 signal     ----> power amplifier Signal in
PXI AO reference         ----> power amplifier signal reference
```

`Signal in` 是 FDEM 功放的模拟波形输入。它不是高低电平触发接口，禁止输入阶跃、矩形波、恒定高电平或任何其他含直流分量的信号。

#### 4.2.2 规定波形

理想的单次发射波形定义为：

```text
u(t) = A sin(2*pi*f*t + phi),  0 <= t < n/f
u(t) = 0 V,                    t >= n/f
```

其中：

- `n`：正弦波完整周期数，必须为正整数；
- `f`：发射频率，单位为 `Hz`；
- `A`：正弦波峰值，当前软件配置为 `6 Vpk`（`12 Vpp`）；
- `phi`：初相位；
- 单次发射持续时间 `T_tx = n/f`；
- 信号的理论平均值必须为 `0 V`。

为了使发射波形在周期边界回到零点，推荐使用 `phi = 0`，并输出整数个完整周期：

```text
voltage
 +A       /\        /\                    /\
         /  \      /  \                  /  \
  0 V --/----\----/----\------ ... -----/----\--
             \  /      \  /                  \  /
 -A           \/        \/                    \/
       |<-------- one period: 1/f -------->|
       |<------------- n periods: n/f ---------------->|
```

> **幅值安全要求：**当前软件配置为 `6 Vpk`（`12 Vpp`），理想波形范围为 `-6 V` 至 `+6 V`。是否满足功放 `Signal in` 输入规格仍需用功放手册和 DC 耦合示波器确认。未经确认不得接入功放。

#### 4.2.3 绝对禁止的信号

以下任何一种信号都不得送入 `Signal in`：

1. 直流信号，包括持续的 `0 V` 以外恒定电平和 TEM 使用的持续高电平。
2. 带直流偏置的正弦信号，例如 `u(t) = A sin(2*pi*f*t) + U_offset` 且 `U_offset != 0`。
3. 未完整结束在零点的截断正弦波。
4. 软件异常、任务结束或设备复位时产生的非零保持电平。
5. DAC 饱和、削顶、跳变或失真的波形。
6. 未经示波器确认幅值、频率、周期数和平均值的波形。
7. 频率、幅值或负载超出功放允许范围的波形。

危险波形示意：

```text
Forbidden DC:

 +U  -------------------------------
  0  _______________________________

Forbidden offset sine:

      /\      /\      /\
 ----/--\----/--\----/--\---------- non-zero center line
    /    \  /    \  /    \
 0 --------------------------------- required zero reference
```

> **损坏后果：**当前 FDEM 功放 IGBT 没有直流、偏置、过流或其他已知保护措施。不能依赖功放自动关断、软件报错、保险丝或操作人员反应来保护 IGBT。危险信号出现后再停止任务已经来不及。

#### 4.2.4 波形生成要求

FDEM 发射软件必须同时满足以下条件：

- 只生成整数 `n` 个完整正弦周期；
- 数字波形以 `0 V` 为中心，不叠加任何偏置；
- 采样率必须足以正确表示频率 `f`，并满足 PXIe-6361 和功放的带宽要求；
- 波形首点、末点及任务停止后的 AO 状态必须经过实测确认；
- 禁止通过输出固定非零电压来等待触发或维持功放状态；
- 禁止复用 TEM 的阶跃波形生成函数；
- 参数非法、数据包含 `NaN`/无穷值、越界或波形均值超限时必须拒绝发射；
- AO 写入、任务启动、任务结束、程序崩溃、SSH 断开和 PXI 重启等场景均须验证不会产生危险电平。

由于实际 DAC 是离散采样系统，波形应按设备支持的时钟生成。若每周期采样点数为 `N_s`，正弦段包含 `n * N_s` 个采样点。为明确回到零点，应在正弦段后追加一个终止采样点 `0 V`：

```text
sample_rate = f * N_s
u[k] = A * sin(2*pi*k/N_s),  k = 0, 1, ..., n*N_s - 1
u[n*N_s] = 0 V
N_total = n * N_s + 1
```

正弦段从首点到追加的零点经历 `n * N_s` 个采样间隔，对应时间 `n/f`。追加零点的目的是避免有限输出任务停在最后一个非零正弦采样值；它不能代替对任务结束后物理端口状态的实测。若使用其他采样率，应确保总时长、周期数和相位边界一致，避免因取整产生非完整周期或终点跳变。

#### 4.2.5 直流偏置检查

软件计算平均值只能作为辅助检查，不能替代示波器测量。应对正弦段的 `n * N_s` 个采样点计算平均值；追加的终止零点不影响结果：

```text
U_mean = sum(u[k]) / (n * N_s),  k = 0, 1, ..., n*N_s - 1
```

要求如下：

- 理论生成值应满足 `U_mean = 0 V`；
- 实际 AO 输出的允许偏置必须依据功放安全规格确定；
- 在该容限未知时，不得自行假设一个“足够小”的安全偏置；
- 每次修改波形算法、采样率、接线或 PXI 配置后，都要重新测量实际偏置；
- 示波器应使用 DC 耦合观察中心电平，不能只用 AC 耦合判断无偏置。

### 4.3 功率输出接口

功率输出连接发射线圈 Tx：

```text
power amplifier power output (+/-) ----> Tx coil terminals
```

功放输出应为与 `Signal in` 对应的正弦波，但其实际电压、电流、相位、增益和失真由功放及负载共同决定，不能只根据 AO 输入推定。

接线和上电前必须确认：

- 功放允许的频率和输入幅值范围；
- 功放增益及最大输出电压、电流和功率；
- 发射线圈的电阻、电感、阻抗随频率的变化及额定电流；
- 功放允许的感性负载范围；
- 功率输出端极性、绝缘和接地方式；
- 电缆、连接器和端子的额定电压、电流及温升；
- 失谐、开路、短路或负载变化对 IGBT 的影响。

`Signal in`、`Start` 和 PXI 通道都不得与功放大功率输出端混接。

### 4.4 电流监测接口

电流监测信号连接到 `ai29`：

```text
Tx current
   |
current sensor / shunt
   |
attenuation and/or isolation
   |
PXIe-4309 ai29
```

`ai29` 用于测量实际发射电流的：

- 频率；
- 幅值；
- 相位；
- 波形失真；
- 是否存在异常直流分量或削顶。

在采样电阻、传感器比例和衰减比未确认前，`ai29` 采集值只能解释为监测电压，不能直接标记为发射电流。

## 5. PXI 模拟输出接口

### 5.1 PXIe-6361 通道分配

| 通道 | 对端 | FDEM 用途 | 关键限制 |
| --- | --- | --- | --- |
| `PXI2Slot3/ao0` | 功放 `Signal in` | 输出 `n` 个频率为 `f` 的正弦周期 | 必须零直流分量，禁止阶跃或非零保持 |
| `PXI2Slot3/ao1` | 功放 `Start` | 输出启动脉冲 | `1 ms` 的 `0 V`，`500 ms` 的 `4 V`，最后回到 `0 V`，不得误接至 `Signal in` |

### 5.2 `ao0` 安全状态

必须使用示波器逐项测量 `ao0` 在以下状态下的实际电压：

- PXI 未启动；
- PXI 启动但程序未运行；
- AO 任务已创建但未启动；
- 正弦波开始前；
- 正弦波输出过程中；
- 正弦波正常结束后；
- AO 任务关闭后；
- 程序异常退出后；
- SSH 连接中断后；
- PXI 板卡复位及系统重启过程中。

“软件写入 `0 V`”“任务关闭”或“设备复位”不等于已经证明物理端口安全。NI-DAQmx 任务结束后端口可能保持最后一个采样值，必须以实测结果为准。

### 5.3 AO 参考地

信号线和参考线必须构成正确回路，但不得在不了解隔离结构时直接把 PXI 模拟地、功放信号地、功率负端、机壳地和保护地短接。

接线前必须：

- 查阅 PXIe-6361、接线盒及功放手册；
- 测量 PXI AO 参考地与功放信号参考地之间的电位差；
- 确认 AO 地线不承载发射功率电流；
- 评估地环路和共模电压；
- 必要时采用经验证且不会引入危险偏置的隔离方案。

## 6. PXI 模拟输入接口

### 6.1 PXIe-4309 通道分配

| 通道 | 信号来源 | 用途 |
| --- | --- | --- |
| `PXI2Slot2/ai29` | 发射电流监测电路 | 测量发射电流波形 |
| `PXI2Slot2/ai31` | 接收线圈或接收前端 | 测量接收信号 |

### 6.2 `ai29` 发射监测通道

`ai29` 应与 `ao0` 的发射过程同步采集，用于确认功放和发射线圈实际产生了预期的正弦电流。重点检查：

- 实际频率是否为 `f`；
- 实际周期数是否与 `n` 对应；
- 波形是否削顶、畸变或发生异常跳变；
- 电流幅值是否在功放、IGBT 和线圈允许范围内；
- 相位是否稳定；
- 是否存在异常平均值。

监测通道不是 IGBT 保护回路。采集到异常后再由软件停止输出，不能视为可靠的实时保护。

### 6.3 `ai31` 接收通道

`ai31` 用于采集接收线圈在频率 `f` 下的响应。接收数据通常用于计算目标响应的幅值和相位。

接线要求：

- 接收线与发射功率线分开布置；
- 优先使用适合现场条件的屏蔽双绞线；
- 根据信号幅值选择前置放大、衰减、限幅和隔离；
- 确认发射耦合瞬态及稳态信号都不超过输入安全范围；
- 明确差分、伪差分或其他端接模式；
- 不得把接收线圈连接到 AO 或功放输出端。

### 6.4 AI 输入安全

- 软件量程不等于硬件绝对最大输入电压；
- 输入不得超过 PXIe-4309、接线盒和前端电路的额定值；
- 高共模电压下必须使用适当的隔离测量；
- AI 参考线不得承载发射电流；
- 未确认端接方式前，不得随意连接 AI 负端、屏蔽层和机壳地。

## 7. 发射与采集时序

### 7.1 推荐逻辑时序

1. 保持功放主电源断开，生成待发射的数字正弦波。
2. 软件检查 `n`、`f`、幅值、总点数、最大值、最小值、平均值和有限数值。
3. 在功放断电且 `Signal in` 断开的条件下测量 `ao0` 实际输出。
4. 确认波形为 `n` 个完整周期、频率为 `f`、幅值符合已确认定义且无可测直流偏置。
5. 确认发射前后 `ao0` 均处于经硬件验证的安全状态。
6. 完成低能量测试配置并连接功放和发射线圈。
7. 启动 AI 采集。
8. 通过 `ao1` 向 `Start` 输出与 TEM 一致的 `0 V -> 4 V` 启动波形，并确认功放已进入准备状态。
9. 输出 `n` 个正弦周期，同时采集 `ai29` 和 `ai31`。
10. 正弦波必须在零点完整结束。
11. 按功放规定时序停止或禁用功放。
12. 保存波形参数、AO 设定值、AI 数据和测试状态。

逻辑时序示意：

```text
time          t0             t1                         t1 + n/f
              |--------------|------------------------------|

AI ai29       [================ continuous acquisition =================]
AI ai31       [================ continuous acquisition =================]

AO ao0        safe zero -----[ n complete sine periods ]---- safe zero
                              ^ frequency f, zero offset

Start         0 V --+ 4 V ----------------------------------------------
                    ^ 0 V -> 4 V startup edge
```

`safe zero` 必须由实际硬件测试确认，不能只根据软件数组推定。

### 7.2 同步要求

若要计算可靠的幅值和相位，AO 与 AI 应使用共享硬件时钟、参考时钟或硬件触发。仅依赖软件依次启动 AI 和 AO 会引入不确定延迟，不适合高精度相位测量。

推荐记录或回采 `ao0` 实际输出，以实际正弦波作为相位参考，而不是只使用软件设定的起始时间。

## 8. 首次接线与调试流程

1. 断开功放主电源、储能电源和发射线圈功率回路。
2. 确认功放型号、端子定义及信号参考端。
3. 确认功放和 AO 端均允许 `6 Vpk`（`12 Vpp`）输入。
4. 确认功放允许频率、输入电压、输入阻抗、偏置容限和负载范围。
5. 确认 PXI 接线盒型号及 `ao0`、`ao1`、`ai10`、`ai29`、`ai31` 的实际端子。
6. 测量各参考地之间的电位差，并确定隔离和接地方案。
7. 保持功放断电，连接示波器到 `ao0` 和 AO 参考端。
8. 分别检查程序未运行、任务启动、正常输出、任务结束、异常退出和设备复位时的波形。
9. 使用 DC 耦合确认 `ao0` 没有直流信号或可测偏置。
10. 确认输出为 `n` 个完整正弦周期，频率为 `f`，起止点没有危险跳变。
11. 在不连接功放时重复多次测试，并覆盖所有计划使用的 `n`、`f` 和幅值。
12. 连接 `Start`、`Signal in`、监测回路和接收回路，再次核对端子及参考线。
13. 使用最低可行母线电压、最低能量和受控负载进行首次带电测试。
14. 同时用独立示波器观察 `Signal in` 和发射电流，不依赖 GUI 单独判断安全。
15. 确认波形、温升、幅值和相位正常后，才可逐步增加测试能量。

> **调试原则：**只要示波器上出现非零恒定电平、中心线偏移、削顶、周期不完整、异常跳变或不明波形，应立即禁止功放上电。若功放已经带电，应使用已验证的硬件断电方式切断能量，不要依赖再次发送软件命令纠正波形。

## 10. 故障与异常处理

### 10.1 `Signal in` 出现直流电平或偏置

如果在功放断电测试时发现直流电平或偏置：

1. 禁止功放上电。
2. 停止 AO 任务并断开 `Signal in` 接线。
3. 检查波形数组是否叠加偏置或使用了错误的 TEM 逻辑。
4. 检查 DAC 量程、接地、端接、任务结束状态和板卡复位状态。
5. 使用 DC 耦合重新测量所有运行状态。
6. 原因查明并完成全部验证前，不得恢复连接。

如果功放已经带电，不得假设软件停止命令能及时保护 IGBT，应立即采用事先验证的硬件断电流程。

### 10.2 正弦波削顶或畸变

检查：

- `6 Vpk`（`12 Vpp`）幅值定义是否与功放手册一致；
- AO 量程或功放输入范围是否不足；
- 采样率是否过低；
- 功放是否超出带宽或输出能力；
- 发射线圈阻抗是否与当前频率匹配；
- 是否存在接地、负载或连接器问题。

### 10.3 发射频率或周期数不正确

检查：

- 采样率与每周期点数的关系；
- 总点数是否为完整周期对应的点数；
- 点数取整是否改变了实际频率；
- NI-DAQmx 实际使用的采样时钟；
- 波形是否在任务结束时被提前截断。

### 10.4 接收相位不稳定

检查：

- AO 和 AI 是否共享硬件时钟或触发；
- 是否用实际回采波形作为相位参考；
- 软件启动延迟是否被误认为相位变化；
- 发射电流相位是否稳定；
- 接收前端、线缆和滤波器是否引入随频率变化的相位延迟。

## 11. TEM 与 FDEM 接口差异

| 项目 | TEM | FDEM |
| --- | --- | --- |
| `Signal in` 用途 | 发射时刻或阶跃控制 | 连续模拟正弦参考输入 |
| 典型输入 | `0 V -> 4 V` 阶跃/保持 | `n` 个完整正弦周期 |
| 频率参数 | 不以稳态正弦频率为核心 | 发射频率 `f` |
| 幅值要求 | 当前 TEM 文档为 `0/4 V` | 当前 FDEM 配置为 `6 Vpk`（`12 Vpp`） |
| 直流分量 | TEM 控制逻辑中存在 | FDEM 严格禁止 |
| 输出场 | 瞬变场 | 频率为 `f` 的交变场 |
| 主要数据 | 关断后的时域衰减 | 幅值和相位响应 |
| IGBT 风险 | 以对应 TEM 硬件设计为准 | 无保护；直流或偏置会立即烧坏 |

TEM 和 FDEM 的 `Signal in` 波形生成逻辑必须分离，不能通过仅修改幅值或持续时间来共用同一阶跃输出流程。
